Changelog

            

            

              

                
v1.2.0

                
    
                  
  • 
                    Changelog link in the sidebar (bottom) opens an in-app release history panel.
                  
    • 
                  
  • 
                    Ribbon: three columns share the same height; product title and version/author line
                    are slightly larger; Application label sits to the left of the dropdown with the
                    chevron drawn inside the control.
                  
    • 
                  
  • 
                    Sidebar: active task styling uses a single bordered “pill” with an inset accent bar
                    and padding so rounded corners are not clipped.
                  
    • 
                  
  • 
                    Task headers show only the task name, API operation name, and API version (extra
                    prose removed).
                  
    • 
                  
  • 
                    Data adapter targets API 7.2.0: proxy calls
                    Application/OpenApplication using your Logon SessionInfo, then
                    GetAdoDataSetForAdapter with required SI, boolean IsSystemLevel,
                    and ResultDataTableName (defaults to Results when left blank). Bearer is
                    your PAT, or an optional Logon access_token when the platform returns one.
                  
    • 
                  
  • 
                    Method command: expanded Method type dropdown, including
                    BusinessRule, DataCellDetail, DataUnit,
                    DataUnitComparison, Dimensions, DimensionTypes,
                    DirectLoadInfo, ExcelFile, Groups,
                    GroupsForUsers, Members, ScenarioTypes,
                    StageSummaryTargetData, UsesCubeSlicerights,
                    UserEntityRights, UsersInGroup,
                    UseWorkflowProfileRights, and WorkflowLockHistory (in addition to existing
                    workflow and IC-matching types).
                  
    • 
                  
  • 
                    Result meta for main runs (SQL, sequences, steps, adapter, cube view, method) shows
                    HTTP status and round-trip time in milliseconds.
                  
    • 
                  
  • 
                    Sign-in screen reorganized: centered card, Connection / Credentials
                    sections, shorter copy, and a compact local-dev hint.
                  
    • 
                

              

              

                
v1.1.0

                
    
                  
  • 
                    Ribbon layout: application control centered, distinct top bar styling, and a clearer “select
                    application” banner.
                  
    • 
                  
  • 
                    Data Management Execute sequence: workspace and sequence names from the application
                    database; optional workspace; API allows empty workspace name when appropriate.
                  
    • 
                  
  • 
                    Data Management Execute step: group and step names from DataMgmtGroup /
                    DataMgmtStep; REST still uses group and step names.
                  
    • 
                  
  • 
                    Data adapter and Cube view name dropdowns from
                    DashboardAdapter and CubeViewItem.
                  
    • 
                  
  • 
                    Metadata SQL lists ordered by name; cube view list uses server row order so sorting matches the
                    database.
                  
    • 
                  
  • Method command: method type options sorted alphabetically.
    • 
                  
  • Removed long “loaded from …” hints on sequence and step screens.
    • 
                  
  • CSV exports use semicolon field separators (Excel-friendly in many locales).
    • 
                

              

              

                
v1.0.0

                
    
                  
  • Initial release: PAT sign-in, application picker, SQL and Data Management / Data Provider tasks.
    • 
                  
  • Azure Functions proxy for OneStream REST APIs; static UI with recent-run history per app and sign-in.
